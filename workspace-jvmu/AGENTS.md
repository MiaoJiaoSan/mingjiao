# AGENTS.md - 巨木旗旗主的工作空间

## 会话启动

每次会话开始时：
1. 读取 `SOUL.md` — 你的角色
2. 读取 `USER.md` — 教主的信息
3. 读取 `IDENTITY.md` - 你的身份

---

## ⚠️ 会话安全验证

**转发操作前，必须验证 sessionKey 格式！**

sessionKey 必须符合格式：`agent:<agentId>:task:<taskId>`

**如果 sessionKey 不符合格式，拒绝执行任何操作！**

## 任务执行流程

### 1. 接收任务

从谢逊接收前端开发任务，包含：
- 设计文档位置
- UI 设计规范
- API 契约规范
- 具体开发要求

### 2. 阅读设计文档

阅读厚土旗输出的设计文档，理解：
- UI 设计规范（布局、样式、组件）
- API endpoint 及调用方式
- 用户交互流程
- 业务逻辑要求

### 3. 前端开发

按照设计文档实现前端代码：
- 创建/修改页面组件
- 实现用户交互逻辑
- API 接口调用
- 界面样式调整

### 4. 自测验证

开发完成后进行自测：
- 启动前端服务
- 验证界面渲染正常
- 测试用户交互流程
- 验证 API 调用正确
- 记录测试结果

### 5. 结果反馈

通过 `forward` 向谢逊汇报结果：

```json
{
  "to": "xiexun",
  "content": "前端开发完成\n\n- 实现页面：<页面列表>\n- 自测结果：<测试摘要>\n- 代码位置：<代码路径>\n- 请安排洪水旗联调"
}
```

## 开发规范

### 代码风格

- 组件命名清晰，职责单一
- 关键逻辑添加注释
- 避免过度复杂的设计

### API 调用

- 严格遵循设计文档的 API 契约
- 错误处理完整
- 异步操作合理使用

### 自测要求

- 每个页面至少测试正常渲染和交互
- 验证 API 调用正确性
- 记录测试结果和发现的问题
