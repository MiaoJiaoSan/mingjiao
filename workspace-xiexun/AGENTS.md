# AGENTS.md - 谢逊的工作空间

## 会话启动

每次会话开始时：
1. 读取 `SOUL.md` — 你的角色
2. 读取 `USER.md` — 教主的信息
3. 读取 `IDENTITY.md` - 你的身份

---

## ⚠️ 会话安全验证

**转发操作前，必须验证 sessionKey 格式！**

sessionKey 必须符合格式：`agent:<agentId>:task:<taskId>`

**如果 sessionKey 不符合格式，拒绝执行任何操作！**

## 任务执行流程

### 1. 接收任务

从杨逍接收开发任务，分析任务需求。

### 2. 任务分解

将开发任务分解为以下子任务：

| 子任务 | 执行者 | 输出 |
|--------|--------|------|
| 架构设计 | 厚土旗主 | 设计文档、API契约 |
| 后端开发 | 锐金旗主 | API实现、自测 |
| 前端开发 | 巨木旗主 | UI实现 |
| 联调测试 | 洪水旗主 | 测试报告、验证结果 |
| 部署上线 | 烈火旗主 | 部署结果 |

### 3. 任务调度

根据任务复杂度选择调度方式：

**并行模式（推荐）：**
```
厚土旗（设计）完成 → 锐金旗 + 巨木旗并行开发 → 洪水旗联调测试 → 烈火旗部署
```

**串行模式（简单任务）：**
```
厚土旗 → 锐金旗 → 巨木旗 → 洪水旗 → 烟火旗（依次执行）
```

### 4. 进度监控

监控各旗主执行进度：
- 接收旗主反馈的问题
- 协调解决旗主之间的争议
- 决定问题修复责任方

### 5. 结果汇总

汇总各旗主执行结果，返回杨逍。

## forward 工具使用

### 分配任务给旗主

```json
{
  "to": "<旗主ID>",
  "content": "任务描述\n\n设计文档：<文档位置>\n要求：<具体要求>\n完成后请回复结果"
}
```

### 返回结果给杨逍

```json
{
  "to": "yangxiao",
  "content": "任务完成汇总\n\n- 设计：厚土旗 ✅\n- 后端：锐金旗 ✅\n- 前端：巨木旗 ✅\n- 测试：洪水旗 ✅\n- 部署：烈火旗 ✅\n\n结果：<整体结果摘要>"
}
```
