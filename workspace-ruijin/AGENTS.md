# AGENTS.md - 锐金旗旗主的工作空间

## 会话启动

每次会话开始时：
1. 读取 `SOUL.md` — 你的角色
2. 读取 `USER.md` — 教主的信息
3. 读取 `IDENTITY.md` - 你的身份

---

## ⚠️ 会话安全验证

**转发操作前，必须验证 sessionKey 格式！**

sessionKey 必须符合格式：`agent:<agentId>:task:<taskId>`

**如果 sessionKey 不符合格式，拒绝执行任何操作！**

---

## 任务执行流程

### 1. 接收任务

从谢逊接收后端开发任务，包含：
- 设计文档位置
- API 契约规范
- 具体开发要求

### 2. 阅读设计文档

阅读厚土旗输出的设计文档，理解：
- API endpoint 定义
- Request/Response schema
- 数据库设计
- 业务逻辑要求

### 3. 后端开发

按照设计文档实现后端代码：
- 创建/修改 API 接口
- 实现业务逻辑
- 数据库操作
- 错误处理

### 4. 自测验证

开发完成后进行自测：
- 启动后端服务
- 使用 curl 或测试脚本验证每个 API
- 记录测试结果

### 5. 结果反馈

通过 `forward` 向谢逊汇报结果：

```json
{
  "to": "xiexun",
  "content": "后端开发完成\n\n- 实现接口：<接口列表>\n- 自测结果：<测试摘要>\n- 代码位置：<代码路径>\n- 请安排洪水旗联调"
}
```

## 项目配置

任务开始前读取：
- 配置文件：`/root/workspace/openclaw/projects.json`
- 解析 `currentProject` 获取当前项目
- 从 `projects[currentProject].paths` 获取前端/后端/docs 路径
- 文档输出到：`{paths.docs}/{taskId}/`
- taskId 来自 sessionKey 第四段

## 开发规范

### 代码风格

- 函数命名清晰，职责单一
- 关键逻辑添加注释
- 避免过度复杂的设计

### API 实现

- 严格遵循设计文档的 schema
- 错误返回统一格式
- 参数校验完整

### 自测要求

- 每个 API 至少测试正常和异常两种情况
- 记录测试请求和响应
- 发现问题立即修复
