## 任务执行流程

### 1. 接收任务

从谢逊接收联调任务，包含：
- 设计文档位置（了解业务流程）
- 后端代码位置（锐金旗输出）
- 前端代码位置（巨木旗输出）
- 后端服务启动方式
- 前端服务启动方式

### 2. 阅读设计文档

阅读厚土旗输出的设计文档，理解：
- 核心业务流程
- API 契约定义
- 用户操作路径

### 3. 启动服务

**启动后端服务：**

```bash
cd <后端代码目录>
# 根据项目类型启动
npm run dev   # Node.js
python app.py # Python
# 或其他启动方式
```

验证后端 API 是否正常：

```bash
curl http://localhost:<端口>/api/<endpoint>
```

**启动前端服务：**

```bash
cd <前端代码目录>
npm run dev
```

验证前端是否正常渲染。

### 4. 执行联调测试

**端到端流程测试：**

按照设计文档的用户操作流程，逐一验证：

1. 打开前端页面
2. 执行用户操作（如：登录、创建、查询等）
3. 验证 API 调用正确
4. 验证界面响应正确
5. 验证数据流转正确

**测试脚本方式（可选）：**

编写集成测试脚本：

```javascript
// integration-test.js
const axios = require('axios');

async function testUserFlow() {
  // 测试创建用户
  const createRes = await axios.post('http://localhost:3000/api/users', {
    name: '测试用户',
    email: 'test@example.com'
  });
  console.log('创建用户:', createRes.data);
  
  // 测试查询用户
  const listRes = await axios.get('http://localhost:3000/api/users');
  console.log('查询用户列表:', listRes.data);
  
  // 验证数据匹配
  if (listRes.data.find(u => u.name === '测试用户')) {
    console.log('✅ 测试通过');
  } else {
    console.log('❌ 测试失败');
  }
}

testUserFlow();
```

### 5. 问题定位

发现问题时，定位责任方：

| 问题类型 | 责任方 | 示例 |
|---------|--------|------|
| API 返回格式不对 | 后端（锐金旗） | 契约定义返回 `{ id, name }`，实际返回 `{ userId, userName }` |
| API 调用失败 | 后端（锐金旗） | 500 错误、数据库连接失败 |
| 界面显示错误 | 前端（巨木旗） | 数据正确但界面渲染有问题 |
| API 调用方式不对 | 前端（巨木旗） | POST 应该用 body，但用了 query |
| 契约理解不一致 | 设计（厚土旗） | 契约描述模糊，前后端理解不同 |

## 测试原则

### 端到端验证

- 不只测单个 API，要测完整用户流程
- 从用户视角验证功能是否可用
- 测试正常流程和异常流程

### 问题定位清晰

- 明确指出问题在哪一方
- 提供详细的错误信息（请求、响应、日志）
- 给出修复建议

### 反复验证

- 修复后必须重新联调
- 直到所有流程通过才能进入部署
- 不放过任何问题
